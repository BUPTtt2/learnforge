# LearnForge 项目交付文档

---

## 一、README

### 1. 项目简介

LearnForge 是一个基于多智能体的AI学习助手系统，旨在帮助用户高效学习新知识。系统采用 Commander-Expert 架构模式，包含讲解师、审查师等多个专业Agent，协同完成学习任务。

### 2. 快速启动

#### 环境要求
- Python 3.8+
- Ollama（本地模型运行）
- Node.js 18+（前端）

#### 安装依赖

```bash
# 安装后端依赖
pip install -r requirements.txt

# 安装前端依赖
cd ui/react
npm install
```

#### 配置环境

```bash
# 复制环境配置模板
cp .env.example .env

# 编辑 .env 文件，填入你的API密钥（如需使用云端模型）
```

#### 启动服务

```bash
# 启动后端服务（端口5000）
python run.py

# 启动前端服务（端口3000）
cd ui/react
npm run dev
```

#### 访问地址

- **前端**: http://localhost:3000
- **后端API**: http://localhost:5000

### 3. 核心API

```bash
# 开始学习
POST /api/learning/start
{
  "knowledge_point": "Python基础",
  "learning_mode": "simple"  # 或 "deep"
}

# 获取学习进度
GET /api/learning/progress/{session_id}

# 获取学习历史
GET /api/learning/history

# 完成学习
POST /api/learning/complete/{session_id}
```

### 4. 验证命令

```bash
# 测试后端服务
curl http://localhost:5000/api/health

# 测试学习API
curl -X POST http://localhost:5000/api/learning/start \
  -H "Content-Type: application/json" \
  -d '{"knowledge_point": "机器学习", "learning_mode": "simple"}'
```

### 2. 核心组件

| 组件 | 职责 | 文件位置 |
|------|------|----------|
| **Commander** | 任务调度与协调 | `agents/commander.py` |
| **Explainer** | 内容生成与讲解 | `agents/讲解师.py` |
| **Reviewer** | 质量审查与反馈 | `agents/审查师.py` |
| **MemoryManager** | 记忆管理（短期/长期） | `memory/memory_manager.py` |
| **KnowledgeGraph** | 知识图谱存储 | `memory/knowledge_graph.py` |
| **CacheSystem** | 语义缓存系统 | `utils/cache.py` |
| **SafetyChecker** | 安全内容审查 | `utils/safety.py` |

### 3. 学习流程

```
用户提问 → Commander接收 → 查询记忆 → 规划章节 → 讲解师生成 → 审查师评估 →
              ↓                                              ↑
              └────────────── 质量不通过 → 自动重做 ─────────┘
                           ↓
                     返回学习内容
```

### 4. 关键设计决策

1. **多Agent架构**：采用 Commander-Expert 模式，职责分离，便于扩展
2. **记忆分层**：短期记忆（会话级）+ 长期记忆（知识图谱）
3. **语义缓存**：基于文本相似度的智能缓存，提升响应速度
4. **安全风控**：输入/输出双重安全检查，防止恶意内容
5. **本地优先**：优先使用Ollama本地模型，保护隐私

## 四、风险提示

### 1. 已知问题

| 问题 | 影响 | 状态 | 计划 |
|------|------|------|------|
| 缺少输入长度限制 | 可能导致DoS攻击 | 待修复 | 后续迭代 |
| 缺少请求频率限制 | 可能被滥用 | 待修复 | 后续迭代 |
| 日志未脱敏 | 敏感信息可能泄露 | 待修复 | 后续迭代 |
| 网络搜索依赖外部API | 需要Tavily密钥 | 可选功能 | 按需启用 |

### 2. 环境依赖

- **Ollama**：必须安装并运行 `ollama serve`
- **模型下载**：首次使用需下载 `qwen3:4b` 或 `qwen2.5:7b`
- **Node.js**：前端构建依赖

### 3. 安全建议

- 不要将 `.env` 文件提交到版本控制
- 定期轮换API密钥
- 生产环境关闭 `FLASK_DEBUG`
- 限制API访问来源
