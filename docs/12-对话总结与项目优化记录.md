## 三、缓存系统改进

### 1. 三大问题解决方案

| 问题 | 解决方案 | 实现方式 |
|------|---------|---------|
| **缓存穿透** | 空值缓存 | `null_cache` 存储不存在的key，5分钟TTL |
| **缓存击穿** | 互斥锁 + 双重检查 | `threading.Lock`，保证只有一个线程查数据库 |
| **缓存雪崩** | 随机TTL波动 | TTL ±20%波动，防止大量key同时过期 |

### 2. 代码改进

**新增方法：**

| 方法 | 功能 |
|------|------|
| `get_with_lock()` | 带互斥锁的缓存查询（防击穿） |
| `add_null()` / `remove_null()` | 空值缓存管理（防穿透） |
| `set()` (改进) | 加入随机TTL波动（防雪崩） |

**扩展统计指标：**
- `penetration_skipped`: 缓存穿透保护次数
- `breakdown_prevented`: 缓存击穿保护次数
- `null_cache_size`: 空值缓存数量

## 五、Agent协作梳理

### 真实架构

**简化后的架构：**

```
Commander（指挥官）
├── 内置 MemoryManager
│   ├── 短期记忆（内存）
│   └── 长期记忆（知识图谱）
├── 内置 ModelManager
│   └── 模型调用 + 缓存
└── 外部调用
    ├── Explainer（讲解师）
    └── Reviewer（审查师）
```

### 学习流程

```
1. 用户请求 → 2. 查询知识图谱（掌握度）→ 3. 规划章节
→ 4. Explainer生成内容 → 5. Reviewer审核 → 6. 更新知识图谱
→ 7. 返回结果
```

## 八、Claude Code Subagent 架构分析

### 1. 设计思路

**核心架构：**
```
主Agent（全局协调）
    ↓ 任务委派
Subagent（专用专家）← 独立上下文 + 受限权限
    ↓ 结果回传
主Agent（合成返回）
```

### 2. Subagent核心特性

| 特性 | 说明 |
|------|------|
| **独立上下文窗口** | 与主对话完全隔离，防止污染 |
| **专用系统提示词** | 每个Subagent有自己的角色设定 |
| **受限工具权限** | 精确控制能使用哪些工具 |
| **独立模型选择** | Haiku/Sonnet/Opus按需分配 |
| **无状态生命周期** | 调用后即销毁，下次重建 |

### 3. 设计优势

| 优势 | 说明 |
|------|------|
| **上下文保护** | 中间数据只留Subagent，不挤占主对话 |
| **行为可控** | 提示词写死约束 + 工具权限限制 |
| **成本优化** | Haiku处理简单任务，Opus处理复杂任务 |
| **并发执行** | 支持多Subagent并行工作 |

### 4. 设计缺点

| 缺点 | 说明 |
|------|------|
| **Token消耗剧增** | ~15x Token消耗增长（Anthropic内部测试） |
| **状态管理复杂** | Subagent之间无共享状态 |
| **调试困难** | 中间过程不透明 |
| **配置成本较高** | 需要多个配置文件 |

### 5. 典型应用场景

- **代码审查**：只读权限 + 质量规范提示词
- **搜索探索**：Haiku模型 + 搜索工具专配
- **重构重构**：深度提示词 + Opus模型
- **测试生成**：测试专用提示词 + 测试工具

## 九、优化建议与讨论

### 1. 语义缓存

**当前问题：** 缓存是精确匹配，需要完全相同的字符串才命中

**改进方案：**
- **快速版：** difflib字符串相似度（10分钟）
- **完整版：** embedding向量相似度（30分钟）

**效果：** "神经网络" 和 "什么是神经网络" 可以命中同一缓存

---

### 2. 学习内容动态调整

**当前问题：** 章节规划是固定的，不会根据练习反馈调整

**改进方案：** 根据练习正确率动态增加/跳过章节
