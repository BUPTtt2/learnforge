## 第2页：项目概述

### 一句话介绍

**"一个基于知识图谱和 RAG 的自适应学习系统，根据用户掌握度动态调整学习内容"**

### 解决什么问题

| 用户痛点 | 解决方案 |
|---------|---------|
| 学习内容千篇一律 | 根据掌握度动态调整难度 |
| 学习进度无法追踪 | 知识图谱追踪掌握度 |
| 重复问题重复问 | 缓存系统加速响应 |
| 生成内容质量不可控 | Reviewer 质量审查 |

## 第4页：技术亮点1 - Agent 协作

### 指挥官-专家模式

| 组件 | 职责 | 特点 |
|------|------|------|
| Commander | 流程编排、任务分配 | 中心协调者 |
| Explainer | 生成学习内容 | 支持简单/深度模式 |
| Reviewer | 质量把控 | 三种决策：approve/rework/retry |
| Memory Manager | 记忆管理 | 短期+长期分离 |

### 重试机制

- Commander：最多重试2次
- Explainer：最多重试2次
- Model Manager：最多重试3次 + 指数退避

---

## 第5页：技术亮点2 - RAG 检索

### 三种检索模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| 本地RAG | Chroma向量库 | 已有知识 |
| 外部RAG | Tavily联网搜索 | 新知识 |
| 混合搜索 | 智能决策 | 复杂查询 |

### 检索优化

- 向量+关键词混合打分（0.6/0.4权重）
- 候选集放大 + 重排序
- 自动评估检索质量

---

## 第6页：技术亮点3 - 知识图谱

### 掌握度追踪

```
掌握度模型：
- 0.0 = 未学
- 0.1-0.7 = 学习中
- 0.8-1.0 = 已掌握

学习流程：
查询掌握度 → 调整学习模式 → 生成内容 → 更新掌握度
```

### 特点

- 树形结构 + 索引加速
- JSON文件持久化
- 支持历史记录追溯

## 第7页：技术亮点4 - 缓存系统 + 风控

### 缓存三大问题解决方案

| 问题 | 解决方案 |
|------|---------|
| 缓存穿透 | 空值缓存 |
| 缓存击穿 | 互斥锁 + 双重检查 |
| 缓存雪崩 | 随机TTL波动 |

### 风控体系

| 层级 | 措施 |
|------|------|
| 输入侧 | 敏感词过滤 + Prompt注入防护 |
| 输出侧 | 违规内容检测 + 轻量清洗 |
