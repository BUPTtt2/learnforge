### 1.2 Commander 内部结构

```
Commander（指挥官）
├── 【内置 Memory Manager】
│   ├── 短期记忆（内存）
│   │   ├── chat_context: 对话上下文
│   │   ├── rag_results: RAG结果缓存（5分钟TTL）
│   │   └── learning_state: 当前学习状态
│   │
│   └── 长期记忆（持久化）
│       └── knowledge_graph: 知识图谱（JSON）
│
├── 【内置 Model Manager】
│   ├── 模型调用：Ollama / 远程API
│   ├── 缓存机制：LRU + TTL
│   └── 负载均衡：多模型切换
│
├── 【外部调用】
│   ├── Explainer.generate_explanation()
│   └── Reviewer.review_all_chapters()
│
└── 【核心功能】
    ├── 流程编排：任务分解与执行
    ├── 状态管理：保存/恢复进度
    └── 重试机制：失败自动重试
```

## 二、完整流程详解

本系统有 **2 种主要流程**：
1. **对话流程**：用户提问 → 系统回答
2. **学习流程**：用户学习知识点 → 系统生成内容 → 审查 → 更新掌握度

---

## 三、流程一：用户对话

### 3.1 涉及模块

| 涉及模块 | 角色 | 具体操作 |
|---------|------|---------|
| **用户交互层** | 发起方 | 用户发送问题 |
| **API 接口** | 入口 | 接收用户请求 |
| **RAG 检索层** | 数据源 | 检索历史对话和知识 |
| **Model Manager** | 生成方 | 调用大模型生成回答 |
| **Memory Manager** | 存储方 | 存储对话上下文 |

### 3.2 详细步骤

```
┌─────────────────────────────────────────────────────────────────────────────┐
│ 第一步：用户发送问题                                                        │
│ 输入：用户通过 API 接口发送问题                                             │
│ 例如："什么是神经网络？"                                                    │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 第二步：API 接收请求                                                        │
│ 模块：api.py                                                                │
│ 操作：解析请求参数，提取问题内容                                            │
│ 数据：question = "什么是神经网络？"                                        │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 第三步：查询 RAG 检索                                                      │
│ 模块：RAG 检索层                                                            │
│ 涉及组件：                                                                 │
│   ├── 聊天历史 RAG → 检索相关历史对话                                       │
│   │   数据：{"content": "之前聊过深度学习的内容", "score": 0.85}            │
│   ├── 外部知识 RAG → 检索外部文档                                           │
│   │   数据：{"content": "神经网络文档片段", "score": 0.92}                  │
│   └── 学习内容 RAG → 检索学习材料                                           │
│       数据：{"content": "神经网络章节内容", "score": 0.88}                  │
│                                                                             │
│ 输出：合并后的检索结果列表                                                  │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 第四步：调用 Model Manager 生成回答                                          │
│ 模块：Commander 内置 Model Manager（hybrid_model_manager）                   │
│ 操作：                                                                     │
│   1. 构建 prompt（包含用户问题 + RAG 检索结果）                            │
│   2. 检查缓存（是否已有相同问题的回答）                                     │
│   3. 缓存未命中 → 调用 Ollama 或远程 API                                    │
│   4. 返回生成的回答                                                        │
│                                                                             │
│ 涉及数据：                                                                 │
│   输入：prompt = "用户问题 + 检索到的上下文"                                 │
│   缓存 Key：hash(问题内容)                                                 │
│   输出：answer = "神经网络是..."                                           │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 第五步：存储对话到短期记忆                                                  │
│ 模块：Commander 内置 Memory Manager                                          │
│ 操作：                                                                     │
│   1. 保存到 chat_context（对话上下文）                                      │
│   2. 保存到 RAG（用于后续检索）                                            │
│                                                                             │
│ 涉及数据：                                                                 │
│   chat_context = {                                                          │
│       "conversation_id": "conv_001",                                        │
│       "messages": [                                                         │
│           {"role": "user", "content": "什么是神经网络？"},                 │
│           {"role": "assistant", "content": "神经网络是..."}                │
│       ]                                                                     │
│   }                                                                         │
│                                                                             │
│ RAG 存储：                                                                  │
│   metadata = {                                                              │
│       "type": "chat",                                                       │
│       "conversation_id": "conv_001",                                        │
│       "timestamp": "2026-05-13T10:00:00"                                   │
│   }                                                                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│ 第六步：返回回答给用户                                                      │
│ 模块：API 接口                                                              │
│ 输出：JSON 格式的回答                                                       │
│ 例如：{"answer": "神经网络是...", "sources": [...]}                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 四、流程二：学习流程

### 4.1 涉及模块

| 涉及模块 | 角色 | 具体操作 |
|---------|------|---------|
| **用户交互层** | 发起方 | 用户发起学习请求 |
| **Commander** | 协调方 | 流程编排、状态管理 |
| **Memory Manager** | 查询方 | 查询知识图谱掌握度 |
| **Explainer** | 生成方 | 生成章节内容和练习 |
| **Reviewer** | 审查方 | 审查内容质量 |
| **Model Manager** | 生成方 | 调用大模型生成内容 |
| **RAG 检索层** | 数据源 | 存储学习内容 |
| **持久化层** | 存储方 | 保存知识图谱 |
