# 架构图生成详细说明

---

## 文档说明

本文档专为画图AI设计，包含详细的架构图生成说明。

**使用方式**：将本文档内容复制粘贴给画图AI，它可以根据描述生成专业的架构图。

## 一、整体架构图 - 四层架构

### 图表类型
- **类型**：分层架构图（Layered Architecture Diagram）
- **风格**：专业技术风格，清晰简洁
- **布局**：垂直分层，从上到下

### 层次结构

#### 第1层：用户交互层（User Interface Layer）
- **位置**：最顶层
- **背景颜色**：浅蓝色（#E3F2FD）
- **组件**：
  - Web UI：圆角矩形，白色背景，黑色文字
  - API 接口：圆角矩形，白色背景，黑色文字
  - 聊天接口：圆角矩形，白色背景，黑色文字
  - 管理后台：圆角矩形，白色背景，黑色文字
- **连接关系**：四个组件并排排列，各自有箭头向下指向第2层

#### 第2层：Agent Teams 协作层（Agent Collaboration Layer）
- **位置**：第2层
- **背景颜色**：浅绿色（#E8F5E9）
- **组件**：
  - Commander（指挥官）：居中放置，圆角矩形，白色背景，粗边框
  - Explainer（讲解师）：左侧，圆角矩形，白色背景
  - Reviewer（审查师）：中间偏右，圆角矩形，白色背景
  - Memory Manager（记忆管理）：右侧，圆角矩形，白色背景
  - Model Manager（模型管理）：底部居中，圆角矩形，白色背景
- **连接关系**：
  - Commander 向下有箭头分别指向 Explainer、Reviewer、Memory Manager
  - Explainer 和 Reviewer 之间有双向箭头
  - Explainer、Reviewer、Memory Manager 向下合并箭头指向 Model Manager
  - Model Manager 向下有箭头指向第3层

#### 第3层：RAG 检索层（RAG Retrieval Layer）
- **位置**：第3层
- **背景颜色**：浅紫色（#F3E5F5）
- **组件**：
  - 聊天历史RAG：圆角矩形，白色背景
  - 外部知识RAG：圆角矩形，白色背景
  - 学习内容RAG：圆角矩形，白色背景
  - 向量数据库：圆角矩形，白色背景，底部居中
- **连接关系**：
  - 三个RAG组件并排排列
  - 三个RAG组件都有箭头指向向量数据库
  - 向量数据库向下有箭头指向第4层

#### 第4层：持久化层（Persistence Layer）
- **位置**：最底层
- **背景颜色**：浅橙色（#FFF3E0）
- **组件**：
  - 知识图谱：圆角矩形，白色背景
  - 用户历史：圆角矩形，白色背景
  - 系统配置：圆角矩形，白色背景
  - 缓存数据：圆角矩形，白色背景
- **连接关系**：四个组件并排排列

### 箭头样式
- **类型**：实心箭头
- **颜色**：深灰色（#37474F）
- **方向**：从上到下，从左到右

## 二、Agent 协作流程图

### 图表类型
- **类型**：流程图（Flow Chart）
- **风格**：清晰的流程导向
- **布局**：从上到下，分支结构

### 流程步骤

1. **用户发起学习请求**
   - 形状：椭圆形，蓝色背景（#1976D2），白色文字
   - 位置：顶部

2. **Commander 接收请求**
   - 形状：圆角矩形，白色背景，黑色文字
   - 位置：步骤1下方

3. **查询知识图谱（Memory Manager）**
   - 形状：圆角矩形，白色背景，黑色文字
   - 位置：步骤2下方
   - 分支：
     - 有记录 → 获取掌握度（菱形判断框）
     - 无记录 → 创建新节点（圆角矩形）

4. **掌握度判断**
   - 形状：菱形判断框，白色背景，黑色文字
   - 位置：步骤3右侧
   - 三个分支：
     - 掌握度 > 0.7 → 深度模式
     - 掌握度 < 0.3 → 简单模式
     - 其他 → 用户指定模式

5. **规划学习章节**
   - 形状：圆角矩形，白色背景，黑色文字
   - 位置：掌握度判断下方

6. **生成章节内容（Explainer）**
   - 形状：圆角矩形，浅蓝色背景（#E3F2FD），黑色文字
   - 位置：步骤5下方

7. **质量审查（Reviewer）**
   - 形状：圆角矩形，浅绿色背景（#E8F5E9），黑色文字
   - 位置：步骤6下方
   - 三个分支：
     - 通过 → 更新掌握度 +0.3
     - 部分重做 → 重试失败章节（循环回步骤6）
     - 全部重试 → 更新掌握度 +0.2

8. **更新知识图谱（Memory Manager）**
   - 形状：圆角矩形，白色背景，黑色文字
   - 位置：步骤7下方

9. **返回结果给用户**
   - 形状：椭圆形，绿色背景（#388E3C），白色文字
   - 位置：底部

### 连接线
- **主线**：实线箭头
- **循环线**：虚线箭头（重试路径）
- **颜色**：深灰色

## 三、Agent 通信关系图

### 图表类型
- **类型**：组件关系图（Component Relationship Diagram）
- **风格**：清晰的连接关系
- **布局**：中心放射状

### 中心组件
- **Commander（指挥官）**
  - 位置：中心
  - 形状：圆形，蓝色背景（#1976D2），白色文字
  - 大小：较大

### 周边组件
- **Explainer（讲解师）**
  - 位置：左侧
  - 形状：圆角矩形，白色背景，黑色文字
  - 连接：实线箭头，Commander → Explainer，标注"同步调用"

- **Reviewer（审查师）**
  - 位置：右侧
  - 形状：圆角矩形，白色背景，黑色文字
  - 连接：实线箭头，Commander → Reviewer，标注"同步调用"
  - 连接：双向箭头，Explainer ↔ Reviewer，标注"数据共享"

- **Memory Manager（记忆管理）**
  - 位置：上方
  - 形状：圆角矩形，白色背景，黑色文字
  - 连接：实线箭头，Commander → Memory Manager，标注"读写操作"

- **Model Manager（模型管理）**
  - 位置：下方
  - 形状：圆角矩形，白色背景，黑色文字
  - 连接：实线箭头，Explainer → Model Manager，标注"模型调用"
  - 连接：实线箭头，Reviewer → Model Manager，标注"模型调用"

### 共享数据
- **知识图谱 JSON**
  - 位置：右上角
  - 形状：文档图标，白色背景，黑色文字
  - 连接：虚线箭头指向所有 Agent，标注"共享内存"

## 四、记忆系统架构图

### 图表类型
- **类型**：层次结构图（Hierarchy Diagram）
- **风格**：树形结构
- **布局**：左右分栏

### 左侧：短期记忆（Short-term Memory）
- **标题**：短期记忆（内存存储）
- **背景颜色**：浅蓝色（#E3F2FD）
- **组件**：
  - 聊天上下文：圆角矩形，白色背景
  - RAG结果缓存：圆角矩形，白色背景
  - 当前学习状态：圆角矩形，白色背景
- **特点标注**：
  - 生命周期：会话级
  - 存储位置：内存
  - 可清除

### 右侧：长期记忆（Long-term Memory）
- **标题**：长期记忆（持久化存储）
- **背景颜色**：浅绿色（#E8F5E9）
- **组件**：
  - 知识图谱：圆角矩形，白色背景
    - 分类节点：圆形节点
    - 知识节点：方形节点
    - 关系边：连接线
- **特点标注**：
  - 生命周期：持久化
  - 存储位置：磁盘
  - 不可清除

### 中间连接
- **双向箭头**：短期记忆 ↔ 长期记忆
- **标注**："同步更新"

## 五、RAG 检索流程图

### 图表类型
- **类型**：数据流图（Data Flow Diagram）
- **风格**：清晰的数据流向
- **布局**：水平流程

### 流程步骤

1. **用户查询输入**
   - 位置：左侧
   - 形状：输入框，白色背景

2. **统一检索接口**
   - 位置：步骤1右侧
   - 形状：圆角矩形，蓝色背景（#1976D2），白色文字

3. **并行检索**
   - 位置：步骤2右侧
   - 三个并行分支：
     - 聊天历史RAG：圆角矩形，白色背景
     - 外部知识RAG：圆角矩形，白色背景
     - 学习内容RAG：圆角矩形，白色背景

4. **结果融合与排序**
   - 位置：步骤3右侧
   - 形状：圆角矩形，紫色背景（#7B1FA2），白色文字

5. **输出结果**
   - 位置：最右侧
   - 形状：输出框，白色背景

### 数据流向
- 箭头方向：从左到右
- 并行检索使用分叉箭头
- 融合使用汇聚箭头

## 六、知识图谱结构图

### 图表类型
- **类型**：知识图谱（Knowledge Graph）
- **风格**：节点-边图
- **布局**：层次树形

### 根节点
- **名称**：分类节点
- **数量**：14个预定义分类
- **形状**：大圆形，蓝色背景
- **示例**：数学、计算机科学、人工智能、机器学习等

### 子节点
- **名称**：知识节点
- **形状**：小方形，白色背景
- **示例**：
  - 数学：微积分、线性代数、概率论
  - 人工智能：神经网络、深度学习、强化学习

### 关系边
- **类型**：父-子关系
- **样式**：实线，灰色
- **标签**："包含"

### 属性标签
- **节点属性**：
  - 掌握度：0.0-1.0
  - 状态：未学/学习中/已掌握
  - 学习历史：时间戳列表

## 七、缓存系统结构图

### 图表类型
- **类型**：分层缓存图（Cache Layer Diagram）
- **风格**：双层架构
- **布局**：上下结构

### 第一层：内存缓存（L1 Cache）
- **位置**：上层
- **背景颜色**：浅蓝色（#E3F2FD）
- **特点**：
  - 大小：小（100项）
  - TTL：1小时
  - 淘汰策略：LRU
  - 访问速度：快

### 第二层：磁盘缓存（L2 Cache）
- **位置**：下层
- **背景颜色**：浅橙色（#FFF3E0）
- **特点**：
  - 大小：大（1000项）
  - TTL：24小时
  - 淘汰策略：LRU + TTL
  - 访问速度：较慢

### 数据流向
- **读取**：先查L1 → 未命中查L2 → 未命中查源数据
- **写入**：同时写入L1和L2
- **箭头**：双向箭头连接两层

## 九、布局建议

### 整体布局
- **方向**：从上到下，从左到右
- **间距**：组件之间保持适当间距
- **对齐**：水平和垂直居中对齐

### 组件大小
- **核心组件**：较大
- **辅助组件**：较小
- **标签**：适当大小，易于阅读

### 连接线
- **避免交叉**：尽量减少线的交叉
- **保持直线**：使用直线连接
- **标注清晰**：重要连接添加文字标注
