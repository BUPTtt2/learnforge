## 缓存三大问题及解决方案

### 1. 缓存穿透

#### 问题描述

缓存穿透是指查询一个一定不存在的数据，由于缓存不命中，每次都去查数据库，导致数据库压力过大。

#### 解决方案：空值缓存

**实现思路：**

1. 当查询的数据不存在时，也缓存一个空值标记
2. 后续查询相同的key时，直接返回None，不查询数据库
3. 空值缓存有过期时间，防止数据更新后无法获取

**代码实现：**

```python
def add_null(self, key: str) -> bool:
    """防穿透：添加空值标记"""
    self.null_cache.add(key)
    self.stats["penetration_skipped"] += 1
    
    # 空值也要有过期时间
    self.set(f"__null_{key}", True, self.null_cache_ttl)
    return True
```

**使用方式：**

```python
cached = cache_manager.get(key)
if cached is not None:
    return cached

# 防穿透检查
if key in cache_manager.null_cache:
    return None

# 查询数据库...
data = db.get(key)

if data is None:
    cache_manager.add_null(key)
else:
    cache_manager.set(key, data)
```

### 2. 缓存击穿

#### 问题描述

缓存击穿是指一个热点key失效，大量请求同时查询这个key，导致数据库压力过大。

#### 解决方案：互斥锁

**实现思路：**

1. 当缓存失效时，只有一个线程去查数据库，其他线程等待
2. 第一个线程查完后，其他线程直接获取缓存
3. 使用双重检查机制，避免重复查询

**代码实现：**

```python
def get_with_lock(self, key: str, loader_func, use_cache: bool = True, ttl: Optional[int] = None) -> Any:
    """带互斥锁的缓存查询（防击穿）"""
    cached = self.get(key, use_cache)
    if cached is not None:
        return cached
    
    # 获取互斥锁
    lock = self.lock_dict[key]
    with lock:
        # 双重检查：获取锁后再次确认
        cached = self.get(key, use_cache)
        if cached is not None:
            self.stats["breakdown_prevented"] += 1
            return cached
        
        # 调用加载函数
        value = loader_func()
        
        if value is None:
            self.add_null(key)
        else:
            self.set(key, value, ttl)
        
        return value
```

**使用方式：**

```python
def load_data_from_db():
    return db.query(key)

data = cache_manager.get_with_lock(key, load_data_from_db)
```

### 3. 缓存雪崩

#### 问题描述

缓存雪崩是指大量key同时过期，导致大量请求同时落到数据库，数据库压力过大。

#### 解决方案：随机TTL波动

**实现思路：**

1. 在设置缓存时，TTL加入随机波动
2. 避免大量key同时过期
3. 保证最小TTL，防止缓存失效过快

**代码实现：**

```python
def set(self, key: str, value: Any, ttl: Optional[int] = None) -> bool:
    if ttl is None:
        ttl = self.default_ttl
    
    # 防雪崩：ttl加入随机波动
    if ttl > 0:
        jitter = int(ttl * self.jitter_factor)
        ttl = ttl + random.randint(-jitter, jitter)
        ttl = max(60, ttl)  # 保证最小ttl为60秒
    
    # ... 其余逻辑
```

## 缓存系统架构

### 双层缓存设计

```
┌─────────────────────────────────────────────────────────┐
│                  Layer 1: Memory Cache                  │
│         (快速，但容量有限，程序重启丢失)               │
│         LRU淘汰，线程安全                           │
└─────────────────────┬───────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────┐
│                  Layer 2: Disk Cache                    │
│        (持久化，容量大，但读取稍慢)                     │
│        JSON存储，启动时自动加载到内存              │
└─────────────────────────────────────────────────────────┘
```

### 缓存Key设计

| 业务场景 | Key格式 | TTL |
|---------|---------|-----|
| 聊天接口 | `chat:MD5(message)` | 1小时 |
| 学习接口 | `learning:MD5(knowledge_point:mode)` | 24小时 |
| 空值标记 | `__null_<key>` | 5分钟 |

## 业务集成方案

### 1. 聊天接口集成

**集成位置：** `/api/chat` 接口

**流程：**

```
用户发送消息
    │
    ▼
检查缓存
    │
    ├─ 命中缓存 → 直接返回，标记 cached: True
    │
    └─ 未命中 → 走原流程生成回答
                      │
                      ▼
              写入缓存（1小时TTL）
                      │
                      ▼
              返回回答，标记 cached: False
```

**代码片段：**

```python
cache_key = f"chat:{hashlib.md5(message.encode('utf-8')).hexdigest()}"
cached_response = cache_manager.get(cache_key)

if cached_response is not None:
    return jsonify({
        'success': True,
        'response': cached_response['response'],
        'cached': True
    })

# 生成回答...

cache_manager.set(cache_key, {
    'response': final_response,
    'tool_results': tool_results
}, ttl=3600)
```

## 面试问题汇总

### Q1: 什么是缓存击穿？如何解决？

**A:**
缓存击穿是指一个热点key失效，大量请求同时查询这个key，导致数据库压力过大。

**解决方案：**
1. 互斥锁：只有一个线程去查数据库，其他线程等待
2. 双重检查：获取锁后再次确认缓存是否已被填充
3. 永不过期：热点数据物理不过期，后台异步更新

**项目实现：**
使用 `threading.Lock` 互斥锁，双重检查机制，统计 `breakdown_prevented` 指标。

---

### Q2: 什么是缓存穿透？如何解决？

**A:**
缓存穿透是指查询一个一定不存在的数据，由于缓存不命中，每次都去查数据库。

**解决方案：**
1. 空值缓存：查不到也缓存一个空值标记
2. 布隆过滤器：提前拦截不存在的请求
3. 接口校验：过滤非法请求

**项目实现：**
使用 `null_cache` 存储空值标记，5分钟TTL，统计 `penetration_skipped` 指标。

---

### Q3: 什么是缓存雪崩？如何解决？

**A:**
缓存雪崩是指大量key同时过期，导致大量请求同时落到数据库。

**解决方案：**
1. 随机TTL：TTL加入随机波动
2. 队列限流：使用消息队列控制并发
3. 多级缓存：本地缓存 + 分布式缓存

**项目实现：**
TTL加入±20%随机波动，保证最小60秒TTL。

---

### Q4: LRU是什么？如何实现？

**A:**
LRU (Least Recently Used) 是最近最少使用淘汰算法。

**实现思路：**
- 哈希表：O(1) 查询
- 双向链表：O(1) 插入/删除
- 每次访问移到链表头部
- 满了从尾部淘汰

**项目实现：**
使用字典模拟，访问时更新 `last_accessed`，淘汰时选择最早访问的。
