---

## 文档说明

本文档专为画图AI设计，包含详细的架构图生成说明，**每个模块都带有深问指向和面试问题**。

---

## 一、整体架构图 - 四层架构

### 图表类型
- **类型**：分层架构图（Layered Architecture Diagram）
- **风格**：专业技术风格
- **布局**：垂直分层

### 层次结构

#### 第1层：用户交互层
- **背景颜色**：浅蓝色（#E3F2FD）
- **组件**：Web UI、API接口、聊天接口、管理后台
- **深问指向**：
  - Q：如何保证接口安全性？
  - Q：聊天接口如何支持流式输出？
- **面试问题**：接口限流策略是什么？

#### 第2层：Agent Teams 协作层
- **背景颜色**：浅绿色（#E8F5E9）
- **组件**：
  - Commander（指挥官）：居中，粗边框
  - Explainer（讲解师）：左侧
  - Reviewer（审查师）：中间偏右
  - Memory Manager（记忆管理）：右侧
  - Model Manager（模型管理）：底部居中
- **连接关系**：
  - Commander → Explainer（标注"同步调用"）
  - Commander → Reviewer（标注"同步调用"）
  - Commander → Memory Manager（标注"读写操作"）
  - Explainer ↔ Reviewer（双向箭头，标注"数据共享"）
  - Explainer、Reviewer → Model Manager（标注"模型调用"）
- **深问指向**：
  - Commander：如何避免单点故障？重试策略如何设计？
  - Explainer：模式差异的具体实现？如何保证内容质量？
  - Reviewer：评分标准如何确定？决策阈值如何设定？
  - Memory Manager：短期/长期记忆边界？如何处理记忆冲突？
  - Model Manager：如何选择本地/远程模型？失败处理机制？
- **面试问题**：为什么采用指挥官-专家模式？如何保证多Agent协作一致性？

#### 第3层：RAG 检索层
- **背景颜色**：浅紫色（#F3E5F5）
- **组件**：聊天历史RAG、外部知识RAG、学习内容RAG、向量数据库
- **连接关系**：三个RAG组件都指向向量数据库
- **深问指向**：
  - Q：向量化模型是什么？
  - Q：结果如何排序和去重？
  - Q：索引重建频率？
- **面试问题**：RAG检索策略是什么？多来源如何融合？

#### 第4层：持久化层
- **背景颜色**：浅橙色（#FFF3E0）
- **组件**：知识图谱、用户历史、系统配置、缓存数据
- **深问指向**：
  - Q：为什么选择JSON而非数据库？
  - Q：备份策略是什么？
- **面试问题**：数据一致性如何保证？缓存和持久化界限？

## 二、Agent 协作流程图

### 图表类型：流程图

### 流程步骤

1. **用户发起学习请求**（椭圆形，蓝色）
2. **Commander 接收请求**（圆角矩形）
   - 深问：状态同步时机如何选择？
3. **查询知识图谱（Memory Manager）**（圆角矩形）
   - 分支：有记录→获取掌握度；无记录→创建新节点
4. **掌握度判断**（菱形）
   - 分支：>0.7→深度模式；<0.3→简单模式；其他→用户指定
   - 深问：掌握度增量如何确定？是否考虑遗忘曲线？
5. **规划学习章节**（圆角矩形）
   - 深问：章节数量和顺序如何确定？
6. **生成章节内容（Explainer）**（圆角矩形，浅蓝背景）
   - 深问：模式差异具体实现？练习题难度如何调整？
7. **质量审查（Reviewer）**（圆角矩形，浅绿背景）
   - 分支：通过→+0.3；部分重做→重试；全部重试→+0.2
   - 深问：评分标准如何确定？决策阈值如何设定？
8. **更新知识图谱（Memory Manager）**（圆角矩形）
9. **返回结果给用户**（椭圆形，绿色）

## 三、Agent 通信关系图

### 图表类型：组件关系图（中心放射状）

### 中心组件
- **Commander**（圆形，蓝色，较大）
  - 面试问题：为什么采用指挥官-专家模式？

### 周边组件
- **Explainer**（左侧）
  - 连接：Commander→Explainer（标注"同步调用"）
  - 深问：如何保证内容多样性？如何避免重复内容？
- **Reviewer**（右侧）
  - 连接：Commander→Reviewer（标注"同步调用"）
  - 连接：Explainer↔Reviewer（双向箭头，标注"数据共享"）
  - 深问：如何避免审查师和讲解师串通？审查结果可解释性？
- **Memory Manager**（上方）
  - 连接：Commander→Memory Manager（标注"读写操作"）
  - 深问：短期记忆容量限制？如何处理并发访问？
- **Model Manager**（下方）
  - 连接：Explainer→Model Manager（标注"模型调用"）
  - 连接：Reviewer→Model Manager（标注"模型调用"）
  - 深问：缓存key如何生成？超时时间如何确定？

### 共享数据
- **知识图谱JSON**（右上角，文档图标）
  - 虚线箭头指向所有Agent，标注"共享内存"

## 七、缓存系统结构图

### 图表类型：分层缓存图（上下结构）

### 第一层：L1内存缓存
- **背景颜色**：浅蓝色（#E3F2FD）
- **特点**：100项，1小时TTL，LRU，快
- **深问**：为什么用LRU？缓存穿透如何处理？

### 第二层：L2磁盘缓存
- **背景颜色**：浅橙色（#FFF3E0）
- **特点**：1000项，24小时TTL，LRU+TTL，较慢
- **深问**：TTL如何平衡性能和一致性？

### 数据流向
- 读取：L1→L2→源数据
- 写入：同时写入L1和L2
- 双向箭头连接两层

### 面试问题
- 缓存和记忆有什么区别？
- 缓存雪崩如何避免？
